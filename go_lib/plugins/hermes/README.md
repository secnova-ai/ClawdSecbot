# Hermes Plugin

**[中文文档](README_zh-CN.md)**

Hermes protection plugin for ClawSecbot.

This plugin integrates [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent) into the repository-wide protection flow, including asset discovery, risk assessment, mitigation, gateway lifecycle hooks, and proxy-based protection.

## Scope

This directory is the plugin implementation for the `Hermes` asset type:

- plugin registration and manifest
- Hermes asset scanning and metadata enrichment
- config-based risk checks
- risk mitigation dispatch and config patching
- gateway restart and config backup/restore
- lifecycle hooks (`OnProtectionStart`, `OnBeforeProxyStop`)
- model connection capability

## Architecture Position

The plugin follows the project standard call path:

`Flutter -> FFI(go_lib/main.go) -> core -> plugins/hermes`

`go_lib/main.go` routes plugin-level runtime settings:

- `SetConfigPathFFI` -> `hermes.SetConfigPath`
- `SetAppStoreBuildFFI` -> `hermes.SetAppStoreBuild`

## Config Discovery

Hermes config path is resolved in this order:

1. explicit override via `SetConfigPath(...)`
2. env `HERMES_CONFIG`
3. env `HERMES_HOME/config.yaml`
4. defaults:
   - `~/.hermes/config.yaml`
   - `~/.hermes/config.yml`
   - `~/.config/hermes/config.yaml`
   - `~/.config/hermes/config.yml`

## Config Fields Used

Current plugin logic reads or writes these fields:

- `model.default`
- `model.provider`
- `model.base_url`
- `model.api_key`
- `terminal.backend`
- `approvals.mode`
- `security.redact_secrets`

## Asset Identity

- asset type: `Hermes`
- `asset_id` is generated by `core.ComputeAssetID(...)`
- identity is stabilized by `config_path` fingerprint to avoid runtime drift from dynamic process/port changes

## Risk Checks

Current risk IDs:

- `config_perm_unsafe`
- `config_dir_perm_unsafe`
- `terminal_backend_local`
- `approvals_mode_disabled` (`off`/`never`/`yolo`)
- `redact_secrets_disabled`
- `model_base_url_public`

Mitigation templates are defined in [`mitigation.json`](./mitigation.json).

## Auto Mitigations

Supported `form` mitigations:

- file permission to `0600`
- directory permission to `0700`
- set `security.redact_secrets=true`
- set `approvals.mode` to `manual` or `smart`

## Protection Lifecycle

### On Start (`OnProtectionStart`)

1. read protection config by `asset_id`
2. load Bot model forwarding config
3. backup Hermes config to:
   - `<backup_dir>/hermes/<sanitized_asset_id>/config.yaml.bak`
4. rewrite Hermes model block to local proxy:
   - `model.provider = custom`
   - `model.base_url = http://127.0.0.1:<proxy_port>`
   - `model.api_key = botsec-proxy-key`
   - `model.default = <bot_model.model>`
5. restart Hermes gateway

### Before Stop (`OnBeforeProxyStop`)

1. restore backup by `asset_id`
2. restart Hermes gateway

## Gateway Restart Strategy

Preferred command:

- `hermes gateway restart`

Fallback when restart fails:

- `hermes gateway stop`
- `hermes gateway start`

The plugin resolves `hermes` from `PATH`.

## App Store Restriction

When `SetAppStoreBuild(true)` is enabled, config rewrite/restart path is blocked and start returns a failure payload by design.

## Testing

Run from `go_lib`:

```bash
go test ./plugins/hermes
go test ./...
```

The plugin directory includes unit tests for scanner/checker/config/mitigation/gateway/proxy session/plugin lifecycle.
